Immich 起動・停止ツール README

このツールは、WindowsメインPC上で Docker Desktop + WSL2 + Ubuntu を使っているImmichを、batファイルのダブルクリックで起動・停止するための補助ツールです。

通常使うのは2つだけ。
Immich起動.bat と Immich停止.bat をダブルクリックして使います。

まずやること

  1. ZIPを解凍します。
  2. フォルダごと C:\Immich に配置します。
  3. Immich起動.bat をダブルクリックします。
  4. 使い終わったら Immich停止.bat をダブルクリックします。
起動 Immich起動.bat

Docker Desktopの起動確認、保存先ドライブ確認、Immich起動、ブラウザ表示まで行います。

停止 Immich停止.bat

Immich関連コンテナを停止し、設定に応じてDocker Desktopも終了します。

同梱ファイル

Immich起動.bat
Immich停止.bat
README.html
src/
├─ Start-Immich.ps1
└─ Stop-Immich.ps1
ファイル 役割
Immich起動.bat 起動用。通常はこれをダブルクリックします。
Immich停止.bat 停止用。使い終わったらこれをダブルクリックします。
src\Start-Immich.ps1 実際の起動処理を行うPowerShellスクリプトです。
src\Stop-Immich.ps1 実際の停止処理を行うPowerShellスクリプトです。

必要な環境

  • Windows
  • Docker Desktop
  • WSL2
  • Ubuntu
  • PowerShell 7
  • Immichの docker-compose.yml 一式

PowerShell 7が入っているか確認するには、PowerShellで以下を実行します。

pwsh -v

環境が違う場合に変更する場所

初期値は「ほどよくPC生活」のImmich導入記事に合わせています。環境が違う場合だけ、src フォルダ内の .ps1 ファイル上部にある設定項目を変更してください。

Start-Immich.ps1

$DistroName = "Ubuntu"
$ImmichDir = "~/immich"
$PhotoPath = "F:\ImmichData\Photos"
$ImmichUrl = "http://localhost:8080"
重要:
導入記事では通常のUbuntuユーザーの ~/immich にImmichを配置しています。 WSLコマンドに -u root を付けると ~/immich が /root/immich を指すため、このツールはWSLの既定ユーザーで実行します。
設定項目 意味
$DistroName WSLのUbuntu名です。wsl -l -v で確認できます。
$ImmichDir Ubuntu内のImmichフォルダです。通常は ~/immich のままでOKです。
$PhotoPath 外部ライブラリとして使う写真・動画フォルダです。導入記事では F:\ImmichData\Photos を使います。
$ImmichUrl ブラウザで開くImmichのURLです。ポートを変えている場合は変更してください。

Stop-Immich.ps1

$DistroName = "Ubuntu"
$ImmichDir = "~/immich"
$QuitDockerDesktop = $true
$ShutdownWsl = $false
設定項目 意味
$QuitDockerDesktop $true ならImmich停止後にDocker Desktopも終了します。Immich以外のDockerコンテナも使っている場合は $false 推奨です。
$ShutdownWsl $true にすると停止時に wsl --shutdown も実行します。他のWSL作業も止まるため、初期値は $false です。

起動時に行う処理

  1. Docker Desktopが起動しているか確認
  2. 起動していなければDocker Desktopを起動
  3. 写真保存先ドライブが見えるか確認
  4. Ubuntu側からDockerが使えるまで待機
  5. docker compose up -d でImmichを起動
  6. immich_server が healthy になるまで待機
  7. ブラウザでImmichを開く

停止時に行う処理

  1. docker compose down でImmich関連コンテナを停止
  2. 設定に応じてDocker Desktopを終了
  3. 設定に応じてWSLを停止
注意:
Docker DesktopでImmich以外のコンテナも動かしている場合は、$QuitDockerDesktop = $false にしてください。 $true のままだと、Docker Desktop自体を終了します。

トラブル時の確認

Ubuntu側でDockerが使えない場合

Docker DesktopのWSL Integrationを確認してください。

Docker Desktop
Settings
Resources
WSL Integration
Ubuntu を ON
Apply & Restart

Ubuntu名が違う場合

PowerShellで以下を実行します。

wsl -l -v

表示名が Ubuntu-24.04 などになっている場合は、$DistroName を同じ名前に変更してください。

Immichが起動しない場合

Ubuntuで以下を実行して状態を確認してください。

cd ~/immich
docker compose ps
docker compose logs --tail=120 immich-server
docker compose logs --tail=120 database

写真保存先が見つからない場合

Start-Immich.ps1 の $PhotoPath が自分の環境と合っているか確認してください。 

$PhotoPath = "F:\ImmichData\Photos"

このツールの想定用途

このツールは、常時稼働サーバーではなく、WindowsメインPCで必要な時だけImmichを使う環境向けです。

専用サーバーで常時稼働させる場合は、Dockerの自動起動やsystemd管理の方が向いている場合があります。

このスクリプトは、WindowsメインPCでImmichを安定して起動・停止するための補助ツールです。 環境によってパス名、Ubuntu名、ポート番号などが異なるため、必要に応じて設定を変更してください。
